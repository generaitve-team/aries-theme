## Directory Purposes

**`.claude/`:**
- Purpose: Claude Code skill configuration and orchestration framework (GSD — Get Shit Done)
- Contains: Agent definitions, command implementations, settings, skill metadata
- Key files: `settings.json`, `package.json`, `gsd-file-manifest.json`
- Note: Generated and maintained by GSD framework; not directly edited by skill development

**`theme/`:**
- Purpose: The complete CSS theme file that implements Aries visual identity
- Contains: Single file — `globals.css` with Tailwind v4 configuration, color tokens, typography utilities, Aries brand colors
- Generated by: Manually crafted (not generated)
- Consumed by: Every Aries project (copied verbatim to `src/app/globals.css`)
- Critical: This is the single source of truth for Aries visual design; always copy as-is, never modify

**`references/`:**
- Purpose: Complete code templates and tutorials for building Aries projects
- Contains: Three markdown files with copy-paste TSX/TypeScript code and step-by-step guides
- Consumed by: Skill implementation (Claude reads and copies code into new projects)
- Key files:
  - `layout-patterns.md`: Complete working app shell, sidebar, header, breadcrumb components
  - `component-guide.md`: Usage patterns for buttons, cards, tables, forms, badges, toasts
  - `scaffold.md`: Step-by-step walkthrough of new project creation

**`scripts/`:**
- Purpose: Automated installation and setup for end users
- Contains: Platform-specific installers (bash for Unix, PowerShell for Windows)
- Responsibility: Check git/Node.js availability, clone repo to `~/.claude/skills/aries-theme/`, update global Claude Code config

## Naming Conventions

**Files:**
- Markdown documentation: `UPPERCASE.md` (SKILL.md, README.md, patterns.md, components.md)
- Code templates: lowercase with hyphens (layout-patterns.md, component-guide.md, scaffold.md)
- Installer scripts: `install.sh`, `install.ps1`
- Theme/CSS: `globals.css` (copied to projects as `src/app/globals.css`)

**Directories:**
- Metadata/config: `.claude/` (hidden directory for Claude Code framework)
- Design assets: `theme/`
- Documentation: `references/`, no separate docs folder
- Automation: `scripts/`

**Configuration Files:**
- `.gitignore` — Standard git ignore
- `.git/` — Standard git directory
- No `tailwind.config.js` or `tailwind.config.ts` — Aries uses Tailwind v4 with CSS-based config in globals.css

## Where to Add New Code

**New Layout Pattern:**
- Add to: `references/layout-patterns.md` under appropriate section (new section if needed)
- Format: Complete TSX code block preceded by explanation of when/why to use it
- Tested: Pattern must be tested in a real Aries project before documenting

**New Component Usage Example:**
- Add to: `references/component-guide.md` under appropriate component section
- Format: Code example showing common usage, variants, and sizing
- Include: When to use, variant/size recommendations, integration with forms if applicable

**New Design Pattern (Prose):**
- Add to: `patterns.md`
- Format: Descriptive section with visual guidance, spacing/color values, examples
- Avoid: Don't add implementation code (that goes in references/)

**New Baseline Component:**
- Add to: `components.md` in the component list with usage note
- Also: Update install command in `SKILL.md`, `components.md`, and `references/scaffold.md` to include new component
- Critical: Test component installation doesn't break existing projects before documenting

**Critical Rules Update:**
- Location: `SKILL.md` under "Critical Rules" section
- When: Only when a new rule affects all Aries projects (e.g., "always use X" or "never use Y")
- Example: "Sidebar brand block is mandatory" or "Tailwind CSS v4 only"

**Skill Installation Logic:**
- Location: `scripts/install.sh` or `scripts/install.ps1`
- When: Installation process needs to change (e.g., new MCP server to register)
- Format: Keep scripts small, focused on environment setup (git, Node.js, file copies)

## Special Directories

**`.claude/`:**
- Purpose: Claude Code framework configuration
- Generated: Automatically by GSD framework
- Committed: Yes, to version control
- Content: Agent definitions, command implementations, skill settings
- Managed by: GSD orchestration system (not manually edited)

**`.git/`:**
- Purpose: Git repository metadata
- Generated: Automatically by `git init`
- Committed: Git internal — always in repo
- Note: This is the aries-theme source repo; users clone this to `~/.claude/skills/aries-theme/`

**`node_modules/`:**
- Purpose: Skill's own dependencies (in `.claude/`)
- Generated: By `npm install`
- Committed: No
- Note: Skills don't typically have dependencies; this may contain GSD framework dependencies

- `src/app/layout.tsx` — Root layout
