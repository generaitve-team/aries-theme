# Technology Stack

**Analysis Date:** 2026-03-20

## Languages

**Primary:**
- TypeScript - Full stack (React components, layouts, utilities)

**Secondary:**
- Bash - Installation scripts and CLI tooling
- PowerShell - Windows installation support

## Runtime

**Environment:**
- Node.js 18+ (detected via `command -v node` in installer, no strict version pinning)

**Package Manager:**
- npm (used exclusively throughout scaffold and component installation)
- Lockfile: Expected but not committed to this repo (project-level only)

## Frameworks

**Core:**
- Next.js 16 - Framework for scaffolded applications (`npx create-next-app@16`)
- React 18+ (via Next.js) - UI components and layouts

**UI Components & Styling:**
- shadcn/ui v4 - Component library (`npx shadcn@4 init`, `npx shadcn@4 add`)
  - Style: new-york
  - Base color: slate
  - Icon library: lucide-react
  - 27 baseline components included

**CSS:**
- Tailwind CSS v4 - CSS-first configuration via `@theme inline` in globals.css (no tailwind.config.js)
- @tailwindcss/postcss - PostCSS plugin for Tailwind v4

**Font:**
- Inter (via `next/font/google`) - Loaded with CSS variable `--font-inter` and weights [300, 400, 500, 600, 700]

**Testing:**
- Not specified in this design system repo (left to implementing projects)

**Build/Dev:**
- Turbopack - Bundler option in Next.js 16 scaffold (`--turbopack` flag)
- PostCSS - CSS processing via `postcss.config.mjs`

## Key Dependencies

**Critical for Aries projects:**
- `shadcn/ui@4` - Provides 27 baseline components (alert, avatar, badge, breadcrumb, button, card, checkbox, collapsible, command, dialog, dropdown-menu, form, input, label, progress, scroll-area, select, separator, sheet, sidebar, skeleton, sonner, switch, table, tabs, textarea, tooltip)
- `lucide-react` - Icons library (Lucide icon system, installed via shadcn)
- `class-variance-authority` - CSS variant utilities
- `clsx` - Class name utility
- `tailwind-merge` - Merges Tailwind CSS classes intelligently
- `sonner` - Toast notification component library
- `tw-animate-css` - Animation utilities for Tailwind CSS (imported as `@import "tw-animate-css"` in globals.css)

**Form handling (required in projects):**
- `react-hook-form` - Form state management
- `zod` - Schema validation for forms

**Utilities (installed in projects):**
- `@tailwindcss/postcss` - PostCSS plugin

## Configuration

**Environment:**
- Project-level environment configuration via `.env` files (not in this repo)
- No external API keys needed for the design system itself
- shadcn MCP server registered in `~/.claude/settings.json` for component lookup

**Build:**
- `postcss.config.mjs` - Must configure `@tailwindcss/postcss` plugin
- `next.config.js` - Generated by create-next-app
- `tsconfig.json` - Generated by create-next-app with path alias `@/*`
- `components.json` - shadcn/ui configuration with new-york style, slate base, CSS variables, @/ aliases

**Root layout configuration:**
- Inter font loaded via `next/font/google` with `--font-inter` CSS variable
- Global stylesheet `src/app/globals.css` must be last write to avoid overwrites by `shadcn add`

## Platform Requirements

**Development:**
- macOS or Linux (installer has specific paths for Mac/Linux)
- Windows via PowerShell with `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser` (one-time)
- git (checked by installer)
- Node.js (checked by installer, offers Homebrew/package manager auto-install)

**Production:**
- Next.js deployment target (Vercel, self-hosted Node.js, Docker, etc.)
- No specific platform lock-in

## Claude Code Integration

**Installation:** Cloned to `~/.claude/skills/aries-theme/` via installation script
- Configured in `~/.claude/CLAUDE.md` with global instruction marker `## Aries Theme Kit`
- shadcn MCP server registered in `~/.claude/settings.json` for real-time component API lookup via `shadcn@latest mcp`
